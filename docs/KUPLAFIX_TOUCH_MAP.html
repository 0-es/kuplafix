  • 00-feature-guide/start here
    • enabled-by-default/works automatically
      • Login welcome summarydefault-on

        After entering the hotel, opens one movable summary with the previous logical visit, populated rooms, every online friend, staff online and cached chat from the previous session. It stores exactly the current and previous visit. Reloading or reconnecting within one hour continues the current visit and cannot replace the previous-visit timestamp; returning after at least one hour advances the pair once. Room cards show Navigator thumbnails and friend heads; friend cards enter that friend’s current room. Friend cards are direct items in one compact three-column grid whose continuous packing path keeps every occupied-room group connected across row changes. Five-pixel group spacing is rendered by an inset background/border layer so every grid item remains the same square size. Groups receive collision-free room tints, while friends without a current room always use neutral grey.

        Menu: Kirjautumisen yhteenveto · Inputs: account/friend/room packets, :userson, Navigator DOM and chat cache · Stores: current/previous logical visit and card position · Controls: staff list, absence threshold and open-now button
      • Online count in the chat boxdefault-on

        Replaces the chat placeholder with the current hotel population and refreshes it at the configured interval. It restores the original placeholder when disabled, prevents overlapping requests and skips checks while the tab is hidden.

        Menu: Kävijämäärä chatboxissa · Network: /api/online-count · Default: 60 seconds · Minimum: 5 seconds
      • Persistent notification bubblesdefault-on

        Lets selected Nitro notifications remain visible instead of disappearing on the native timer. KuplaFix clones persistent bubbles, can add timestamps, de-duplicates repeated content and supports click or shift-click dismissal.

        Menu: Säilytä ilmoitukset · Types: friend status, achievements, badges, gifts, respect and currencies
      • Chat history and session cachedefault-on

        Replaces the native history button with a searchable KuplaFix window and optional popout. It preserves rendered chat across logins, inserts session separators, supports system/bot/WIRED filters and highlights the last message from the previous session.

        Menu: Chat-historia · Default storage: 1,000 messages for 7 days · Welcome excerpt excludes system, bot and WIRED messages
      • Compact event invitationsautomatic fix

        Hides the intrusive hotel-event window and recreates its title, image and room action as a compact notification. The selector matches Nitro’s separate nitro-alert-hotel and event classes.

      • Update indicatormenu-triggered

        Checks the GitHub userscript version when the KuplaFix menu opens and turns the subtitle into an install link when a newer version exists. It does not check on page load and is throttled to once per minute.

    • optional-mods/off until enabled
      • GIF controlsopt-in

        Handles GIF chat in three real modes: spoiler click-to-reveal, allow only selected usernames, or hide every GIF message. The menu manages the username whitelist and now uses the correct configuration key.

        Menu: Piilota gifit · Modes: spoiler, allow-selected, block-all · Observer exists only while enabled
      • Room lightingopt-in

        Applies brightness and warm night-light filtering directly to room canvases. Sliders and presets adjust brightness, sepia, hue and saturation; filters are reapplied when Nitro replaces the canvas.

        Menu: Huonevalaistus · Observer exists only while enabled · Changes canvas filter and transition styles
      • Game modeopt-in

        Turns WASD into room movement or one-character quick chat. Other letters and numbers can be sent immediately, while a temporary toggle above chat pauses the mode without changing the saved main setting.

        Menu: Game mode · Controls: WASD behavior and temporary hotkey · Captures keydown/keyup in both documents
      • Voice messagesopt-in

        Adds a press-and-hold or double-click recorder beside chat. Audio is resampled and encoded into a KuplaFix payload or uploaded through the configured bridge; other KuplaFix clients render playback chips with waveform progress.

        Menu: Ääniviestit · Permission: microphone · Uses MediaRecorder and AudioContext · Scans only new chat subtrees
      • LiveKit meetingsopt-in

        Adds a sidebar button for a draggable embedded LiveKit meeting. It requests a short-lived token and permits camera, microphone, screen sharing, autoplay and clipboard write inside the meeting iframe.

      • In-game Kupla front pageopt-in

        Adds a home button and opens the Kuplahotelli front page in a draggable sandboxed overlay without leaving the game. The page can also be moved into a separate browser window.

    • power-tools/advanced
      • Renderer and UI config overridesopt-in

        Accepts JSON overrides for Nitro’s renderer-config.json and ui-config.json. KuplaFix wraps fetch and XHR, changes only matching responses and returns untouched responses without issuing a duplicate request.

      • Packet Builder, filters, logger and historyadvanced

        Captures the Hanarchy WebSocket, names known headers, displays bounded incoming/outgoing history while the builder is open, parses typed fields and can send or inject constructed packets. Persisted header filters can block either direction or rebuild selected parsed arguments while preserving every unlisted live value. Visual logging is persisted separately and is off by default.

      • Packet macros and sharingadvanced

        Runs conditional send, inject, toast and delay actions for matching packets. Enabled macros are indexed by direction/header instead of scanning every macro for every packet. KFM2: macros can be shared through chat or trophies with a trusted-sender safeguard.

    • kuplafix-framework/support layer
      • Settings interface

        Adds the toolbar/sidebar button, draggable translucent menu, ordered and collapsible feature cards, automatic saves, status text, toasts and reset controls.

      • Userscript and bookmarklet startup

        Prevents duplicate instances, runs from the top document, injects shared styles and rebinds modules whenever the same-origin Nitro iframe changes. Bookmarklet mode preserves the complete SSO URL while restarting the client for early socket hooks.

      • Public integration API

        Exports window.kuplafix, window.kuplaSocket, typed packet helpers, listener/interceptor APIs, configuration access and update-test helpers for debugging and companion mods.

  • 01-implementation-touch-map/technical reference
    • 00-bootstrap/always1–594
      • userscript-metadatadocument-start

        Matches *://kuplahotelli.com/game/nitro*, declares @noframes, and requests GM value/XMLHTTP plus window focus/close grants. The first runtime guard exits inside frames, so the userscript itself starts in the top document.

      • bookmarklet lifecyclespecial path

        Consumes window.__KUPLAFIX_BOOKMARKLET_BOOT__, suspends Nitro DOM callbacks, reloads the same-origin #nitro iframe with its original SSO URL, then installs the socket hook in the fresh window. Packet interception is explicitly described in source as deterministic only in document-start userscript mode.

      • singleton and exported globalsalways

        Reads/writes window.__kuplafix_singleton__. Exports window.kuplafix (version, config, UI, cache, welcome, packet tools and test helpers) and window.kuplaSocket after capturing the game socket.

      • GM fallbackbookmarklet

        If GM APIs are absent, stores values as JSON in same-origin localStorage using the prefix kuplafix_gm_, and emulates GM_xmlhttpRequest with fetch.

      • DOM lifecycle brokeralways

        Touches the top document and same-origin #nitro iframe. It observes iframe replacement, rejects about:blank, exposes cross-document query/wait/inject helpers, and replays callbacks when a real Nitro document becomes available.

        Primary surfaces: document, document.body, document.documentElement, #nitro, iframe.contentDocument, iframe.defaultView

    • 03-feature-controllers/mixed activation3853–7661, 12159–13810
      • ChatInputObserveralways

        Mutation-observes top/Nitro documents for chat input creation, then reattaches online count, voice controls and game-mode toggle. One observer is retained per document.

      • ChatEnhancements / online countdefault-on

        Calls /api/online-count, writes “N hahmoa paikalla” into the native chat placeholder, suppresses overlapping requests, and repeats every configured interval (60 seconds by default).

      • UpdateCheckeron menu open

        Fetches the raw GitHub userscript with a timestamp query, extracts @version, caches the result and turns the menu subtitle into an install link when newer. Page-load checks are intentionally disabled.

      • GifBlockeropt-in

        While enabled, Mutation-observes both documents, debounces rescans by 150 ms, finds Nitro GIF bubbles and supports whitelist, reveal-placeholder, non-whitelisted hide, or hide-all behavior. Disabling restores hidden content and disconnects its observers.

      • RoomLightingopt-in

        Finds room canvases and applies CSS brightness, sepia, hue-rotate and saturate filters. Its 120 ms debounced observer exists only while the feature is enabled.

      • LiveKitopt-in

        Adds a sidebar entry, reads the logged-in username, fetches a token URL from the configured worker, and embeds meet.livekit.io/custom in an iframe allowed to use camera, microphone, display capture, autoplay and clipboard-write.

      • BubbleAlertsdefault-on

        Observes native right-side notification bubbles. Depending on per-type auto-despawn settings, it timestamps native bubbles or clones them into #kuplafix-notification-clones, hides the original, de-duplicates matching content for ten seconds, and supports click/shift-click dismissal.

      • ChatHistoryUIdefault-on

        Intercepts native history controls during document capture in both the top and Nitro documents, so React can replace the button without creating a native-history race. The handler recognizes the current Finnish title, English title variants and the legacy room-tools icon; disabling the feature leaves Nitro’s native click untouched. It creates a custom history window, filter controls and an optional window.open popout, copies source styles into the popout and mirrors search/system/bot/WIRED filters.

      • ChatHistoryCachedefault-on

        Observes direct children of the native chat widget, captures newly rendered bubbles, rejects near-duplicates, caps the array, saves after a two-second debounce, restores cached bubbles, inserts session dividers and briefly highlights the final message from the previous login.

      • VoiceMessagesopt-in

        Adds a press/hold/double-click record control. Uses getUserMedia, MediaRecorder, AudioContext, resampling/ADPCM-like text encoding, optional worker upload, incremental added-subtree scanning, object URLs, playback chips and animation-frame progress.

      • EventInviteBlockeralways

        Watches #draggable-windows-container, hides matching event invitation windows and recreates their text/image/action as compact KuplaFix notifications.

      • Browseropt-in

        Adds a sidebar button and draggable overlay containing a sandboxed same-origin iframe. Closing navigates it to about:blank; opening defaults to the Kuplahotelli front page; popout uses window.open.

      • GameModeopt-in

        Captures keydown/keyup in both documents. WASD either clicks/dispatched arrow controls or sends one-character chat messages. It blocks normal single-character input while active and adds a temporary toggle above chat.

      • LoginWelcomedefault-on

        Listens to account/friend/room packets, sends :userson, suppresses its generic-alert response, parses room membership, closes visible MOTD responses, reads Navigator thumbnails and friend-bar avatars, stores current/previous logical visits, and renders the movable once-per-login summary. A saved activity gap under one hour resumes the current logical visit; a gap of one hour or more moves it to the previous slot. Migration collapses the former reload snapshot chain into the same two-slot model. Friend cards are positioned directly in a compact three-column grid. Its continuous cell path and coordinate-based neighbor lookup keep each room blob connected and merge only true neighbors. Group gaps change pseudo-layer insets instead of margins, preserving identical card geometry for top-only, left-only and combined boundaries. It allocates one stable tint per occupied friend-room group and explicitly applies a grey palette to roomless friends.

    • 05-websocket-and-packets/global hook7662–12158
      • WebSocket replacementalways

        Replaces window.WebSocket and patches the original prototype's send. Only URLs containing hanarchy.net are captured. Indexed header filters run before the game/server sees a packet; accepted packets then go through macro/listener/interceptor machinery. Bounded hexadecimal history is allocated only while Packet Builder is open.

      • operational packet IDsdirect use
        IN 2725 USER_INFO — logged-in account for welcome state IN 3130 MESSENGER_FRIENDS — initial friend roster IN 2800 MESSENGER_UPDATE — friend changes IN 758 ROOM_ENTER — triggers automatic welcome flow once IN 3801 GENERIC_ALERT — intercepted for :userson response; may be suppressed OUT 1314 UNIT_CHAT — :userson and optional shared-macro chat message OUT 2312 ROOM_ENTER — welcome-card room navigation OUT 3492 CATALOG_PURCHASE — optional bronze trophy containing a shared macro
      • packet cataloguereference data

        The builder contains 468 outgoing and 469 incoming symbolic names. Merely cataloguing an ID does not send or intercept it. The packet builder and user-created macros can, however, send/inject arbitrary headers and typed arguments.

      • header filtersuser-driven

        Enabled filters are indexed by direction:header. A block rule suppresses the matching binary packet. A modify rule runs the Builder parser on each live packet, replaces only configured zero-based argument indexes, rebuilds the packet with typed values and forwards the result. Incoming synthetic replacement events bypass the filter once to prevent recursion.

      • macros and sharinguser-driven

        Macros trigger on matching packet direction/header, parse arguments, evaluate comparisons, substitute {{argN}}, then send, inject, delay or show a toast. Shared KFM2: payloads are detected in chat and trophy/furniture data; acceptance defaults to the hardcoded trusted sharer unless “accept all” is enabled.

    • 06-browser-platform-apis/capabilities
      • observation and scheduling

        Uses many MutationObserver instances, plus setTimeout, setInterval, requestAnimationFrame, queueMicrotask, page lifecycle events and DOMContentLoaded. There are no Web Workers or IndexedDB paths.

      • media

        navigator.mediaDevices.getUserMedia, MediaRecorder, AudioContext/webkitAudioContext, Audio, Blob, FileReader, and object URLs are used by voice messages and packet Blob decoding.

      • user interaction

        Captures mouse, pointer, touch, keyboard, drag-and-drop, input/change, form submit, window message, iframe load, clipboard write, legacy document.execCommand('copy'), popup windows and new tabs.

      • rendering

        Does not draw to Canvas directly; it changes native room canvas CSS filters. It heavily injects DOM/CSS and clones native Nitro nodes. Avatar/thumbnail pixels are sourced from Kupla endpoints and Navigator DOM.

    • 08-attention-points/know the limits
      • Renderer overrides are startup-sensitivereload required

        Changing renderer/UI JSON in the menu cannot retroactively change configuration Nitro already fetched. Save the setting and reload; document-start userscript mode gives the earliest and most reliable hook.

      • Bookmarklet socket timing is best-effortknown limit

        The bookmarklet safely restarts the same-origin iframe and preserves SSO parameters, but deterministic interception of the very first WebSocket still belongs to userscript document-start mode.

      • Nitro class names remain external contractsmaintenance

        KuplaFix integrates with native DOM classes and packet formats from an abandoned client. A server/client markup change can break a feature without a JavaScript syntax error.

      • Performance improvements are code-verified, not benchmarkedmeasurement

        The removed downloads, intervals and full-document scans are directly verifiable. The repository still lacks a repeatable cold/warm load benchmark against production.
